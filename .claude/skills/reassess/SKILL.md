# Reassess

`$ARGUMENTS` — path to a project directory.

Read everything: `plan.md`, `decisions.md`, `research/research-context.md`, `research/problem-repository.md`, `research/problem-selection.md`. Grep the library for any newly ingested papers relevant to the project.

Answer these questions honestly:

1. **Has someone solved this?** Check if newly ingested papers close the gap we identified.
2. **Is the gap still where we thought?** Has the literature shifted the interesting question?
3. **Is there a sharper version?** Could we narrow or reframe for more impact?
4. **Is there a bigger version?** Did we discover something more ambitious that's now tractable?
5. **Are there better problems?** Check `research/problem-repository.md` — has anything surfaced that's more compelling?

Write your assessment to `decisions.md`. Recommend one of:
- **Continue** — the problem is still right, proceed to next phase
- **Sharpen** — revise the problem statement (explain what to change)
- **Pivot** — switch to a different problem (explain why and to what)
- **Abandon** — the problem is solved or not tractable (explain why)
